# Roles and Responsibilities

This document defines the core roles and access responsibilities established within the Nutrition Assistant application.

---

## System Roles

### 1. Customer / Regular User
A general consumer using the application to plan and track their nutritional targets.
*   **Responsibilities:**
    *   Creating and securing their personal profile credentials.
    *   Inputting and maintaining accurate personal body metrics (weight, height, age, activity level).
    *   Selecting primary goals (e.g., weight loss, weight gain, maintenance).
    *   Reviewing daily calorie and macronutrient targets.
    *   Tracking and implementing the recommended meal templates.

### 2. Dietitian
A specialized user role designed to offer dietary reviews and menu customization options.
*   **Responsibilities:**
    *   Accessing consenting user profiles and logs to review nutrition targets.
    *   Recommending customized meal plans and recipes.
    *   Adjusting macronutrient proportions for specific requirements or conditions.
    *   Offering professional guidance on diet schedules and lifestyle adjustments.

### 3. System Administrator
The coordinator managing operations, configuration settings, and data audits.
*   **Responsibilities:**
    *   Ensuring server environments, database connections, and API routes are healthy.
    *   Managing platform user accounts and auditing roles (User, Dietitian, Admin).
    *   Overseeing global calculations and validation constraints.
    *   Analyzing usage metrics and database performance parameters.
